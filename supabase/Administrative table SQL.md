# Administratrive table Setup

To set up admin table, run the following SQL in your Supabase SQL editor:

```sql
-- Ensure required extension for gen_random_uuid()
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- Create schema if missing (public usually exists)
CREATE SCHEMA IF NOT EXISTS public;

-- Create the admins table
CREATE TABLE IF NOT EXISTS public.admins (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text UNIQUE NOT NULL,
  created_by text,
  created_at timestamptz DEFAULT now()
);

-- Optional: add a comment describing the table (adjust as desired)
COMMENT ON TABLE public.admins IS 'Administrators table with unique email, creator, and timestamps';

-- Optional: add comments on columns
COMMENT ON COLUMN public.admins.id IS 'Primary key, uuid generated by gen_random_uuid()';
COMMENT ON COLUMN public.admins.email IS 'Unique admin email address';
COMMENT ON COLUMN public.admins.created_by IS 'Identifier for the creator (text)';
COMMENT ON COLUMN public.admins.created_at IS 'Timestamp when the row was created (timestamptz)';

-- Recommended: ensure indexes (email is unique so index exists). If you want an explicit index:
-- CREATE UNIQUE INDEX IF NOT EXISTS idx_admins_email ON public.admins(email);

-- If you prefer uuid_generate_v4() from uuid-ossp instead of gen_random_uuid(), use:
-- CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
-- and change DEFAULT gen_random_uuid() to DEFAULT uuid_generate_v4();
```
After running this SQL, the administrative table feature will be fully functional!
